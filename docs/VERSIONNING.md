🏷️ Versioning

Format :
MAJOR.MINOR.PATCH

Exemple :
0.1.0

MAJOR : changements incompatibles (rare avant la v1).
MINOR : nouvelles fonctionnalités.
PATCH : corrections de bugs.

Version	Contenu
v0.1.0	Fondations (architecture, Git, documentation, providers, SearchEngine)
v0.2.0	MatchingService
v0.3.0	ScoringService
v0.4.0	Premier provider réel
v0.5.0	Notifications
v0.6.0	Bot Telegram fonctionnel
v0.7.0	Cache + optimisation
v0.8.0	Tableau de bord Web
v0.9.0	Version bêta
v1.0.0	Premier MVP public
